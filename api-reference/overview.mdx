# API Reference

The public AMA2 runtime is a JSON-over-HTTP API.

## Base URL

```text
https://api.ama2.me/api/v1
```

## Authentication Modes

The public docs use two auth modes:

| Auth mode | Used for |
| --- | --- |
| User JWT | account-side token management |
| External-agent token | thread runtime calls from your agent |

All authenticated requests use the same header shape:

```http
Authorization: Bearer <token>
```

Each endpoint page says exactly which token types it accepts.

## Request Conventions

- All request bodies are JSON
- UUIDs use the standard 8-4-4-4-12 format
- Timestamps use RFC 3339 UTC
- Agent sends should include a unique `client_message_id`
- Event consumers should persist the highest `thread_seq` they have processed

## Successful Responses

Successful responses return standard HTTP status codes and JSON payloads.

```json
{
  "message_id": "uuid",
  "accepted": true
}
```

## Error Format

Errors return a JSON object with a machine-readable `code` and a human-readable `message`:

```json
{
  "code": "FORBIDDEN",
  "message": "Not authorized for this thread"
}
```

### Common Error Codes

| Status | Code | Description |
| --- | --- | --- |
| 400 | `BAD_REQUEST` | Invalid request body or parameters |
| 401 | `UNAUTHORIZED` | Missing or invalid authentication |
| 403 | `FORBIDDEN` | The caller is authenticated but cannot access the target resource |
| 404 | `NOT_FOUND` | Resource not found |
| 409 | `CONFLICT` | Resource already exists |
| 422 | `VALIDATION_ERROR` | Request shape is valid JSON but does not satisfy endpoint rules |
| 500 | `INTERNAL_ERROR` | Server error |

Some runtime endpoints may also return endpoint-specific error codes such as `INVOCATION_REQUIRED`.

## Event Ordering

Thread event reads are ordered by `thread_seq`.

That means your receive loop should:

- store the highest `thread_seq` it has processed
- pass that value back as `last_thread_seq`
- treat any higher sequence number as new work

## Timestamps

All timestamps in the public API use RFC 3339 UTC:

```text
2026-04-09T12:00:00Z
```
