## List Calendar Events

Returns a paginated list of calendar events for the authenticated user.

### Authentication

<Note>Requires a valid user JWT token.</Note>

### Query Parameters

<ParamField query="start" type="string">
  Start of date range (RFC 3339 format).
</ParamField>

<ParamField query="end" type="string">
  End of date range (RFC 3339 format).
</ParamField>

### Response

Returns an array of calendar event objects:

<ResponseField name="id" type="string">UUID of the event.</ResponseField>
<ResponseField name="title" type="string">Event title.</ResponseField>
<ResponseField name="description" type="string">Event description (nullable).</ResponseField>
<ResponseField name="location" type="string">Event location (nullable).</ResponseField>
<ResponseField name="start_at" type="string">Start time (RFC 3339).</ResponseField>
<ResponseField name="end_at" type="string">End time (RFC 3339).</ResponseField>
<ResponseField name="all_day" type="boolean">Whether it's an all-day event.</ResponseField>
<ResponseField name="status" type="string">`"confirmed"` or `"pending"`.</ResponseField>
<ResponseField name="source" type="string">Origin: `"user"`, `"agent"`, `"external_request"`, or `"system"`.</ResponseField>
<ResponseField name="attendees" type="array">List of attendees.</ResponseField>
<ResponseField name="is_recurring" type="boolean">Whether the event recurs.</ResponseField>

### Example

```bash
curl "https://api.ama2.me/api/v1/calendar/events?start=2026-04-01T00:00:00Z&end=2026-04-30T23:59:59Z" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```
