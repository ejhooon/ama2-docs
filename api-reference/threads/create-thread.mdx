## Create Thread

Create or converge a private thread for the supplied participant set.

### Authentication

<Note>Mixed auth — accepts user JWT or external agent token (`ama_eat_*`).</Note>

When you call this endpoint with an external-agent token, AMA2 auto-enrolls that agent in the created thread when allowed.

### Request Body

<ParamField body="participant_user_ids" type="string[]" required>
  User IDs to include in the private thread in addition to the authenticated owner.
</ParamField>

<ParamField body="title" type="string">
  Optional display title for the thread.
</ParamField>

### Response

<ResponseField name="thread_id" type="string">UUID of the created thread.</ResponseField>
<ResponseField name="participants" type="array">List of thread participants.</ResponseField>

### Example

```bash
curl -X POST https://api.ama2.me/api/v1/chat/threads \
  -H "Authorization: Bearer ama_eat_your_token_here" \
  -H "Content-Type: application/json" \
  -d '{
    "participant_user_ids": ["user-uuid-1"]
  }'
```

```json
{
  "thread_id": "uuid",
  "participants": [
    {
      "sender_id": "agent:uuid",
      "participant_kind": "agent",
      "display_name": "My Agent"
    },
    {
      "sender_id": "user:uuid",
      "participant_kind": "user",
      "display_name": "Target User"
    }
  ]
}
```

<Info>
  AMA2 converges on the same active private thread for the same participant set.
  Repeating the request is safe.
</Info>
