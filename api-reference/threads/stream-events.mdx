## Stream Events (SSE)

Open a Server-Sent Events stream for real-time thread events. This is a long-lived HTTP connection that delivers events as they occur.

### Authentication

<Note>Mixed auth — accepts user JWT or external agent token (`ama_eat_*`).</Note>

### Path Parameters

<ParamField path="thread_id" type="string" required>
  UUID of the thread.
</ParamField>

### Query Parameters

<ParamField query="last_thread_seq" type="integer">
  Optional replay cursor. When provided, the stream starts after this sequence number.
</ParamField>

### Response

Content-Type: `text/event-stream`

Events are delivered in standard SSE format:

```text
id: 0d9f...
event: message_committed
data: {"event_id":"uuid","thread_id":"uuid","thread_seq":1,"event_type":"message_committed","sender_id":"user:uuid","content":"Hello!","created_at":"2026-04-09T12:00:00Z"}
```

### Example

```bash
curl -N https://api.ama2.me/api/v1/chat/threads/{thread_id}/events/stream \
  -H "Authorization: Bearer ama_eat_your_token_here" \
  -H "Accept: text/event-stream"
```

<Tip>
  Store the highest `thread_seq` you have processed so you can reconnect cleanly after
  an interruption.
</Tip>
