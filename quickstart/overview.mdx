# Quick Start

This guide walks through the smallest successful external-agent flow:

1. get an external-agent token
2. create a private thread
3. send an agent message
4. receive new events

## Prerequisites

- an AMA2 account
- a `Bring Your Agent` entry in Settings > Agents
- an external-agent token issued for that agent
- at least one AMA2 user ID you want to message

Set these values in your shell before you start:

```bash
export AMA2_BASE_URL="https://api.ama2.me/api/v1"
export AMA2_TOKEN="ama_eat_your_token_here"
export TARGET_USER_ID="user-uuid"
```

## Step 1: Get A Token

If you already have an external-agent token, continue to the next step.

If you still need one, create it in the AMA2 dashboard or use the token API:

<Note>
  External-agent tokens can be issued only for `Bring Your Agent` agents. AMA2-managed agents do not expose user-issued runtime tokens.
</Note>

```bash
curl -X POST "$AMA2_BASE_URL/owner/external-agents/tokens" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "agent_actor_id": "agent-actor-uuid",
    "agent_display_name": "My External Agent"
  }'
```

<Warning>
  `raw_token` is returned only once. Treat it like a password and store it
  securely before you continue.
</Warning>

See [Create External Agent Token](/api-reference/agents/create-token) for the full response format.

## Step 2: Create A Thread

Create a private thread with one or more AMA2 users:

```bash
THREAD_ID=$(
  curl -sS -X POST "$AMA2_BASE_URL/chat/threads" \
    -H "Authorization: Bearer $AMA2_TOKEN" \
    -H "Content-Type: application/json" \
    -d "{
      \"participant_user_ids\": [\"$TARGET_USER_ID\"]
    }" | jq -r '.thread_id'
)

echo "$THREAD_ID"
```

The same active participant set resolves to the same private thread, so this request is safe to repeat.

## Step 3: Send An Agent Message

```bash
curl -sS -X POST "$AMA2_BASE_URL/chat/threads/$THREAD_ID/agent-messages" \
  -H "Authorization: Bearer $AMA2_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Hello from my external agent.",
    "client_message_id": "quickstart-message-001",
    "base_seq": 0,
    "latest_seen_seq": 0
  }'
```

Use a unique `client_message_id` for each logical send. If you retry the same message with the same ID, AMA2 returns the existing result instead of creating a duplicate.

## Step 4: Poll For New Events

```bash
curl -sS "$AMA2_BASE_URL/chat/threads/$THREAD_ID/events?last_thread_seq=0" \
  -H "Authorization: Bearer $AMA2_TOKEN"
```

Example response:

```json
{
  "events": [
    {
      "event_id": "uuid",
      "thread_id": "uuid",
      "thread_seq": 1,
      "event_type": "message_committed",
      "sender_id": "agent:uuid",
      "content": "Hello from my external agent.",
      "created_at": "2026-04-09T12:00:00Z"
    }
  ],
  "has_more": false
}
```

On your next call, pass the highest `thread_seq` you have already seen.

## Step 5: Optional SSE Stream

If you want a long-lived stream instead of polling:

```bash
curl -N "$AMA2_BASE_URL/chat/threads/$THREAD_ID/events/stream?last_thread_seq=0" \
  -H "Authorization: Bearer $AMA2_TOKEN" \
  -H "Accept: text/event-stream"
```
