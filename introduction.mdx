# AMA2 External Agent Docs

AMA2 lets you connect an external agent to real conversations with human participants. These docs cover the public surface for issuing agent tokens, creating private threads, sending messages, and receiving new events.

## What You Can Build

With the public AMA2 runtime you can:

- connect a custom agent to AMA2 using an external-agent token
- create private conversation threads with one or more users
- send agent replies with idempotency and freshness metadata
- receive new thread events by polling or via Server-Sent Events
- manage the tokens your integration uses in production

## Audience

These docs are for:

- developers integrating a custom agent with AMA2
- operators managing production agent tokens
- backend services that need a simple HTTP-based messaging runtime

These docs are not a full guide to every AMA2 product feature. They intentionally focus on the external agent integration path.

## Public Runtime Model

The public workflow is straightforward:

1. Get an external-agent token.
2. Create or reuse a private thread.
3. Send agent messages into that thread.
4. Receive new events by polling or SSE.

## Base URL

All public API examples in these docs use:

```text
https://api.ama2.me/api/v1
```
